# GraphCaster — план разработки

Порядок ориентирован на **короткие инкременты**, каждый из которых оставляет репозиторий в **собираемом/запускаемом** состоянии (Python CLI / UI).

**Сводка:** см. `PRODUCT_DESIGNE.md` — в т.ч. **папка `graphs/`** (автоскан, автосохранение без диалога, открыть из меню и инспектора), **уникальный `graphId`**, **артефакты только под root-run**, **очистка + учёт диска по графу**, **запрет правки при активном Run**, **locale по умолчанию = системная**.

## Архитектура: несколько параллельных графов

- Каждый активный прогон = отдельная **Run-сессия** с собственным **контекстом** (переменные, ссылки на артефакты, состояние) и потоком событий.
- **MVP / отладка:** несколько сессий в **одном** процессе Python (реестр `run_id → состояние`) — проще логировать и тестировать. **Сделано в Python:** `RunSessionRegistry`, `get_default_run_registry()`, опция **`GraphRunner(..., session_registry=…)`**, кооперативная отмена **`request_cancel`**, убийство подпроцесса **`task`** при отмене, CLI **`run --track-session`**, **`--control-stdin`**, **`--run-id`**; см. `python/graph_caster/run_sessions.py`, `process_exec.py`.
- **Целевой режим после интеграции с Cursor CLI:** опция **отдельного дочернего процесса на run** (изоляция падений нативного CLI, перезапуск без гибели UI). Координация: **stdio NDJSON** или локальный сокет; один «координатор» может пулить воркеры.
- **Планировщик в продукте (первая волна): не делаем.** Фоновый Run без открытого редактора — допустим по команде/внешнему скрипту позже.

## Фаза 1 — Контракт JSON v1.1 (Start / Exit / meta)

- Расширить схему: типы нод **`start`**, **`exit`**, (существующие **`task`**, **`graph_ref`** или эквиваленты).
Инварианты: ровно одна **`start`**, без входящих рёбер в **`start`**; **`exit`** — терминальные по смыслу успеха.
- **`meta`:** `author` (строка), **`graphId`** (стабильный UUID; **уникален** в workspace — проверка при save/rename).
- Обновить `graph-document.schema.json`, пример, тесты, `GraphDocument.from_dict`. **Сделано в Python:** `validate_graph_structure`, `GraphRunner.run()` и завершение на **`exit`** + событие **`run_success`**; CLI без **`-s`** использует валидацию и **`start`**.
- Рёбра с условиями: **рантайм — порядок в массиве `edges` в JSON, первое истинное**; **UI — предупреждение** (жёлтый блок + ⚠), **без** блокировки save.
- **Workspace (MVP):** модель «корень проекта»: папка **`graphs/`** (имя по умолчанию) — единственное место **сохранения**; индекс **graphId → путь** строится **автосканом**; дубликат **graphId** — **ошибка сохранения**.

## Фаза 2 — Runner: Start→Exit, однопоточный MVP

- Старт только от **`start`**; завершение при **первом** достигнутом **`exit`** (событие `run_success` / `run_end`).
- Сохранить защиту от бесконечного шага (лимит итераций / детектор цикла).
- Контекст выполнения: словарь/объект для результатов нод и пользовательских переменных; сериализация в событиях **без** огромных бинарей (только ключи/метаданные/пути).
- **`graph_ref`:** целевой **`graphId`** + загрузка документа через **реестр workspace** (маппинг guid→путь); по умолчанию **синхронное** ожидание вложенного Run.
- **Сделано в Python (часть фазы 2):** `workspace.py` (скан **`*.json`**, дубликат **`graphId`** — ошибка, кэш индекса); `GraphRunner` с опцией **`graphs_root`**, события **`nested_graph_enter`/`nested_graph_exit`**, контекст **`node_outputs`**, **`nesting_depth`**, **`max_nesting_depth`**; CLI: подкоманда **`run`** (**`--graphs-dir`** / **`-g`**, **`--artifacts-base`**), обратная совместимость **`graph-caster -d …`**; **`artifacts-size`**, **`artifacts-clear`**.
- **Артефакты:** для **корневого** Run — **`runs/<root_graph_guid>/<run_timestamp>/`**. Вложенные **`graph_ref`:** те же физические записи в **корневую** папку этого Run; в контексте выполнения прокидывать **`root_run_artifact_dir`** (или эквивалент) при рекурсивном вызове.
- **Сделано в Python:** при **`artifacts_base`** в контексте (CLI **`--artifacts-base`**) на старте корневого run создаётся **`{artifacts_base}/runs/<graphId>/<UTC-datetime>_<uniq>/`**, в контексте **`root_run_artifact_dir`**; событие **`run_root_ready`**; во вложенном **`nested_graph_enter`** поле **`rootRunArtifactDir`** для той же строки пути.
- Утилиты: подсчёт **размера на диске** по **`runs/<graph_guid>/`** для UI настроек; действие **«Очистить артефакты»** (область уточнить: весь `runs/` или по выбранному графу).
- **Сделано в Python:** **`tree_bytes`**, **`artifacts_tree_bytes_for_graph(artifacts_base, graph_id)`**, **`artifacts_runs_total_bytes(artifacts_base)`**; **`clear_artifacts_for_graph`**, **`clear_all_artifact_runs`**; защита **`graph_id`** от разделителей и **`..`**.
- **Ноды с процессом (Cursor CLI):** конфиг `successMode` (exit_code | stdout | marker_file), **timeout**, **kill**, **retry** + backoff — в MVP.
- **Сделано в Python (MVP подпроцесса):** нода **`task`** с полем **`command`** или **`argv`** (строка или список); **`cwd`**; **`env`** (дополнение **`os.environ`**); **`successMode`**: **`exit_code`**, **`stdout`**, **`marker_file`**; **`timeoutSec`**, **`retryCount`** / **`maxRetries`**, **`retryBackoffSec`**; события **`process_*`**; при ошибке после попыток — останов run (как **`graph_ref`**).
- Условия на рёбрах: этап **1** — булев контекст + простые предикаты; этап **2** — каталог операций строк/чисел; этап **3** — **ИИ-ветвление** по контракту **request/response JSON** (результат ноды + пронумерованные исходящие связи с описаниями → ответ с **целым индексом** выбранной связи); адаптер провайдера + мок в тестах.

## Дорожная карта: фиксы проблемных мест (по ревью и оценке готовности)

Короткие инкременты; каждый шаг — **собираемый** стенд (Python + UI). Привязка к фактам: паритет JSON/UI/Python уже покрыт тестами (`pytest`, Vitest, `npm run build`); ниже — **продукт**, **UX** и **процесс**.

### P1 — Ошибки при открытии / невалидный JSON

- **Симптом:** `alert` без указания, *что* именно отклонил `parseGraphDocumentJson` / `JSON.parse`.
- **Цель:** одно понятное сообщение (тип ошибки + по возможности путь/ключ: например «`nodes` must be array», «invalid `schemaVersion`»); строки через **i18n**.
- **Гейт:** ручная проверка: испорченный файл → сообщение помогает исправить без чтения кода.
- **Затрагивает:** `ui/src/layout/AppShell.tsx` (и при необходимости тонкий слой поверх `parseGraphDocumentJson`).
- **Сделано:** модальное окно **`OpenGraphErrorModal`** + **`presentationForParseError`** / **`presentationForJsonSyntaxError`** / **`presentationForReadFailure`** (`ui/src/graph/openGraphErrorPresentation.ts`); ключи **`app.errors.openModal.*`** (en/ru); при открытии из файла или из **`graphs/`** в заголовке — **`titleWithFile`** с именем файла; копирование деталей в буфер. См. [`IMPLEMENTED_FEATURES.md`](IMPLEMENTED_FEATURES.md) (раздел про открытие графа).

### P2 — Прозрачность `sanitizeGraphConnectivity` при экспорте

- **Симптом:** рёбра на несуществующие ноды **тихо** отбрасываются при `exportDocument`.
- **Цель:** если кардинальность `edges` уменьшилась — **неблокирующее** уведомление (баннер/строка в **ConsolePanel** или toast): число удалённых рёбер, опционально список `id` (ограничить длину).
- **Гейт:** сценарий: удалить ноду с виспящим ребром → Save → пользователь видит, что граф «подчистился`.
- **Сделано:** `sanitizeGraphConnectivity` возвращает `{ document, removedEdgeIds }`; `GraphCanvas` → `onExportRemovedDanglingEdges`; полоса предупреждений в `AppShell` + i18n **`app.editor.removedDanglingEdges`**; Vitest **`ui/src/graph/sanitize.test.ts`**. См. [`IMPLEMENTED_FEATURES.md`](IMPLEMENTED_FEATURES.md).

### P2 — CI монорепозитория для GraphCaster

- **Сделано:** в корне монорепо **Aura** — **`.github/workflows/graph-caster-ci.yml`**: на **push**/**pull_request** в **`main`** при изменениях под **`third_party/graph-caster/**`** (и при правке самого workflow) — **Ubuntu**, Python **3.11**, **`pip install -e ".[dev]"`** + **`pytest -q`** в **`third_party/graph-caster/python`**; **Node 20.19**, **`npm ci`**, **`npm test`**, **`npm run build`** в **`third_party/graph-caster/ui`** (локально — **`engines`** в **`ui/package.json`**, **`>=20.19`**). Отдельно **`graph-caster-desktop.yml`** остаётся для сборки Windows-инсталляторов (**`workflow_dispatch`**). Указатель в **`docs/AUTOTESTS_CATALOG.md`** §**4.3**.
- **Гейт:** красная сборка при падении любого из шагов job **test**.

### P3 — Расхождение `schemaVersion` в файле (редкий hand-edit)

- **Симптом:** в JSON могли различаться корень и `meta.schemaVersion`; после **Save** выравнивается, но до сохранения редактор показывает «как в файле».
- **Цель:** одно предупреждение в блоке статуса (рядом с предупреждениями структуры), если после парса оба поля присутствуют и численно различаются.
- **Гейт:** один пример JSON с `schemaVersion: 1` и `meta.schemaVersion: 2` → видим ⚠ один раз до первого сохранения.
- **Сделано:** `findStructureIssues` → **`schema_version_mismatch`** (не блокирует Run); `comparableSchemaVersions` в `parseDocument.ts`; i18n **`app.structure.schemaVersionMismatch`**; Vitest **`structureWarnings.test.ts`**, **`parseDocument.test.ts`**.

### P3 — Автосохранение в workspace при ошибке записи

- **Симптом:** в `AppShell` ошибка `writeJsonFileToDir` при autosave глотается (`catch` пустой для «transient» ошибок).
- **Цель:** минимум — счётчик/флаг «последняя автосохранение не удалась» + одно сообщение в UI или консоль (без спама: debounce показа).
- **Гейт:** имитация отказа записи (только dev) → пользователь видит сигнал.
- **Сделано:** баннер **`app.editor.autosaveFailedBanner`** в полосе предупреждений; строка в консоль **`app.editor.autosaveFailedConsole`** не чаще **1 раза / 30 с**; сброс флага при успешном autosave или **Save** в workspace (`saveDocumentToWorkspace`).

## Критерии готовности MVP

- В UI: полотно, **start** (одна) и **exit**, **task**, **graph_ref** со ссылками-кнопками; пины/условия (минимум простой уровень); мультивыбор, undo/redo, поиск, контекстное меню с поиском, мини-карта; консоль с фильтрами/экспортом/очисткой.
- В Python: Run от **start** до **exit**, однопоточно, события стабильны; контекст данных между нодами (хотя бы примитивы + ссылки на артефакты).
- **Целевой сценарий MVP:** воспроизводимая цепочка **автоматизации через Cursor CLI** (подача промптов, ветвление по простому условию).
